# Code of Conduct

## Our Commitment

OpenCourses is a learning community. Everyone who participates — learners, contributors, maintainers — deserves a respectful, harassment-free experience regardless of age, background, disability, ethnicity, gender identity, level of experience, nationality, personal appearance, race, religion, or sexual identity.

We are committed to making participation a positive experience for everyone.

---

## Expected Behaviour

- **Be respectful.** Disagreements happen. Critique ideas, not people.
- **Be constructive.** Feedback on content or code should help the recipient improve.
- **Be patient.** Many contributors are learners themselves. Meet people where they are.
- **Be honest.** Attribute sources correctly. Do not claim others' work as your own.
- **Be inclusive.** Use language that is welcoming to newcomers and people from diverse backgrounds.
- **Ask before assuming.** Clarify intent before escalating a misunderstanding.
- **Review with care.** Explain the risk, cite the file or behavior, and suggest a practical next step.
- **Respect learning pace.** A beginner question is not a burden; it is part of the project.

---

## Unacceptable Behaviour

The following will not be tolerated in any project space:

- Harassment, intimidation, or personal attacks
- Discriminatory jokes, language, or imagery
- Publishing private information (doxxing) without consent
- Sexual attention or advances of any kind
- Deliberate misgendering or deadnaming
- Sustained disruption of discussions or reviews
- Brigading, pile-ons, or coordinated harassment
- Any conduct that would reasonably be considered inappropriate in a professional setting

---

## Scope

This Code of Conduct applies to all project spaces managed by this repository: GitHub Issues, pull requests, Discussions, code review comments, and any other channels linked from this repository. It also applies when an individual is publicly representing the project.

It applies equally to code contributions, course content, review threads,
assignment feedback, leaderboard/certificate discussions, and social links
published by the project.

---

## Review Culture

OpenCourses is both software and education. Reviews should protect learners
without making contributors feel small.

- Point to concrete behavior, content, or code.
- Separate correctness issues from style preferences.
- Ask for sources when factual claims are unclear.
- Give extra context when a request touches accessibility, security, privacy, or licensing.
- Avoid sarcasm, status games, or dismissive shorthand.

---

## Reporting

If you witness or experience unacceptable behaviour, report it privately:

**GitHub:** Open a [private security advisory](https://github.com/manojpisini/opencourses/security/advisories/new) (we use this channel for both security and conduct reports).

Reports are reviewed by the project maintainer within **48 hours**. All reports are kept confidential. Reporters will not face retaliation.

---

## Enforcement

Confirmed violations result in one or more of the following, at the maintainer's discretion:

| Severity | Action |
|---|---|
| Minor, first offence | Private written warning |
| Repeated or moderate | Temporary suspension from project spaces |
| Severe or persistent | Permanent ban |

Decisions are final. Appeals may be sent to the same private channel.
