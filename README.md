## вңҰ What Is OpenCourses?

**The first developer education platform built entirely on GitHub infrastructure.**

OpenCourses is not a MOOC. It is not a wiki. It is not a docs site behind a login wall.

It is a living curriculum вҖ” where course content lives in Git repositories,<br/>
learning progress is tracked in issues and pull requests,<br/>
and every update ships exactly like a software release.

Certificates are GPG-signed GitHub Releases.<br/>
Grades are run in a hardened Docker sandbox.<br/>
The leaderboard rebuilds itself every night.

<br/>

**No servers. No subscriptions. No lock-in. No friction.**

## вңҰ Why It Exists

| The old way | The OpenCourses way |
|:-----------:|:-------------------:|
| Content locked behind accounts | Everything public, on GitHub |
| No version history вҖ” errors persist silently | Every change is a commit, every fix is a PR |
| Black-box grading you can't inspect | Open-source Docker sandbox grader |
| Certificates you can't verify | GPG-signed GitHub Releases вҖ” cryptographically provable |
| Stale courses with no accountability | Maintainer listed, version tagged, status tracked |
| Platform disappears вҶ’ learning gone | Fork the repo вҶ’ nothing lost |
| Expensive subscriptions | Free. Always. No exceptions. |

## вңҰ How Learning Works

```
  1. ENROLL в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     Open a course page and choose Enroll.
     Use the web form, copy the gh command, or copy the generated body.
     The bot processes it instantly. Your student journey begins.

  2. LEARN в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     Read Markdown content in this repo.
     Watch linked videos. Write code locally. No tracking. No timers.

  3. SUBMIT в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     For code assignments: open a Pull Request "[Assignment] @username вҖ” course/ch-01"
     GitHub Actions runs your code in a hardened Docker sandbox.
     Score + line-by-line feedback posted as a PR comment in ~45 seconds.

  4. QUIZ в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     For chapter and final tests: use the course page submission action.
     Prefer the web form, gh command, or copied body path you like.
     The grader checks your answers against the course answer key automatically.
     Feedback posted within minutes. No manual review for standard tests.

  5. ADVANCE в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     Pass a chapter? The bot labels your enrollment issue automatically.
     Fail? Feedback posted. Iterate. No penalty. No lock-out.

  6. CERTIFY в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
     Submit the final test. Pass? A GPG-signed certificate is issued
     as a GitHub Release. Your name goes on the public leaderboard.
     The certificate lives on GitHub forever.
```

## вңҰ Adding a Course

> **All you need to touch is `engine/courses/`.**

### 1 вҖ” Copy the template

```bash
cp -r engine/courses/template engine/courses/your-course-slug
```

### 2 вҖ” Edit `course.yaml`

Open `engine/courses/your-course-slug/course.yaml` вҖ” a pure YAML file with 18 sections:

```yaml
metadata:
  id: your-course-slug            # permanent, kebab-case
  version: "1.0.0"
  status: draft                   # draft | review | published | archived
  solutions_file: solutions.yaml  # answers companion (never commit to main)

identity:
  title: "Your Course Title"
  tagline: "One compelling sentence."
  description:
    short: "Shown in course cards вҖ” under 200 characters."
    full: |
      Full multi-paragraph Markdown description.

classification:
  category: web                   # one of 12 track slugs (see /docs/creators)
  level: beginner                 # beginner | intermediate | advanced | mixed
  tags: [tag1, tag2]

people:
  curator:
    name: manojpisini
    github: manojpisini
    role: Curator & Maintainer

# ... chapters, chapter_tests, assignments, certificate, changelog ...
# See engine/courses/template/course.yaml for the full annotated template.
```

### 3 вҖ” Open a PR

```bash
git checkout -b course/your-course-slug
git add engine/courses/your-course-slug/
git commit -m "course: add Your Course Title"
git push -u origin course/your-course-slug
# Open a PR вҶ’ validate-pr.yml checks the schema вҶ’ merge вҶ’ live in ~90 seconds
```

On merge: `course-publish.yml` parses `course.yaml` вҶ’ writes `course.json` вҶ’ `sync-site-data.yml` aggregates вҶ’ `deploy-site.yml` rebuilds the site.

**You never manually edit `site/src/data/*.json` вҖ” they are machine-generated.**

**Ground rules**

1. Write for practitioners вҖ” assume the student has shipped something before
2. No paywalled resources вҖ” all exercises and readings must be freely accessible
3. Own your course вҖ” keep it accurate, use `status: "attention"` when you need help
4. One PR per course вҖ” don't bundle unrelated changes
5. Be honest about state вҖ” `difficulty: "beginner"` with `status: "draft"` is fine

## вңҰ Tech Stack

| Layer | Technology | Notes |
|:-----:|:----------:|:-----:|
| Site framework | [Astro 4](https://astro.build) вҖ” `output: static` | Zero JS framework shipped to browsers |
| Interactivity | Vanilla JS IIFEs | Catalog filter, force-directed graph, вҢҳK palette |
| Styling | Custom CSS design system | ~42 KB total, no Tailwind, no utility classes |
| Fonts | DM Sans + JetBrains Mono + Sora | Google Fonts, loaded from CDN |
| Knowledge graph | Custom force simulation | No D3 вҖ” pure physics in ~150 lines of JS |
| Hosting | GitHub Pages | Free, org-native, zero-config after setup |
| CI/CD | GitHub Actions | Build, deploy, grade, certify вҖ” all in one place |
| Grading | Docker on `ubuntu-latest` | Reproducible, isolated, deterministic вҖ” no AI |
| Certificates | GPG-signed GitHub Releases | Cryptographically verifiable, permanently public |
| Data pipeline | Bun + TypeScript scripts | Aggregates engine output вҶ’ site JSON automatically |

<br/>
